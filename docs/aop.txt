شرح AOP في المشروع

AOP تعني فصل المهام المشتركة عن منطق الخدمات الأساسي

في هذا المشروع نستخدم Interceptor عام لقياس زمن تنفيذ طلبات HTTP

اسم الكلاس هو PerformanceInterceptor

يعمل قبل تنفيذ الطلب وبعد انتهائه

قبل التنفيذ يسجل وقت البداية

بعد التنفيذ يحسب المدة بالميلي ثانية

ثم يكتب سجل يحتوي على معلومات مفيدة عن الطلب

المعلومات التي يتم تسجيلها

نوع الطلب مثل GET أو POST

المسار المطلوب

اسم Controller

اسم الدالة التي نفذت الطلب

حالة الاستجابة

مدة التنفيذ

رقم المستخدم إذا كان مسجلا للدخول

رقم نسخة التطبيق التي نفذت الطلب

وقت التنفيذ

نوع الخطأ إذا فشل الطلب

لا يتم تسجيل البيانات الحساسة

لا يتم تسجيل كلمة المرور

لا يتم تسجيل access token

لا يتم تسجيل payment token

يتم تسجيل الطلبات البطيئة والأخطاء بشكل واضح حتى يسهل اكتشاف المشكلة

يمكن التحكم بالسجلات من خلال متغيرات البيئة

PERFORMANCE_LOG_ALL=true

هذا الخيار يجعل النظام يسجل كل الطلبات

PERFORMANCE_WARN_MS=500

أي طلب يتجاوز هذه المدة يعتبر بطيئا

بعض الطلبات لا تدخل في مراقبة الأداء لأنها تستخدم للمراقبة نفسها

GET /health

GET /instance

GET /resources

مهام BullMQ والمهام المجدولة لا تمر داخل طلب HTTP لذلك لا يقيسها هذا Interceptor

يتم متابعة هذه المهام من خلال حالة المهمة وسجلات Worker
